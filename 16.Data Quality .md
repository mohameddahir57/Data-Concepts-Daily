##  **What Is Data Quality?**

Data Quality means **how good, reliable, and trustworthy the data is**.

High-quality data helps companies:

 Make accurate decisions
 Avoid mistakes
 Understand customers
 Build better products
 Improve performance

Low-quality data causes confusion, wrong analysis, and money loss.



##  The 3 Core Dimensions of Data Quality

##  1. Accuracy ‚Üí Is the data correct?

Accuracy means the data reflects **the truth**.

Examples of accurate vs inaccurate data:

| Scenario      | Accurate    | Inaccurate       |
| ------------- | ----------- | ---------------- |
| Customer age  | 25          | 250              |
| Product price | $19.99      | $1999 by mistake |
| Phone number  | Real number | Wrong digits     |

Inaccurate data leads to wrong analysis.



##  2. Completeness ‚Üí Is anything missing?

Completeness means **all important values are present**.

Examples:

* Missing customer phone numbers
* Missing product names
* Missing dates in sales data
* Missing email addresses

Incomplete data makes analysis weak because you don‚Äôt have the full picture.



##  3. Consistency ‚Üí Does the data match across all systems?

Consistency means the data is the **same everywhere**.

Examples:

* In one system: Customer name = ‚ÄúMohamed Osman‚Äù
* In another system: ‚ÄúMohamed O.‚Äù

Or:

* App shows product price = $20
* Database shows price = $25

This creates confusion and destroys trust.



##  Real-Life Example of Bad Data Quality

Imagine a company sending emails to customers.
If data is wrong:

* Names spelled incorrectly
* Emails missing
* Duplicate entries
* Inconsistent records

The emails might fail, and the company loses money and trust.



## üõ† How Data Analysts Improve Data Quality

Data analysts use tools and techniques to ensure data is clean and reliable.

###  1. Data Validation

Check if data looks correct (e.g., no negative ages).

###  2. Removing Duplicates

Make sure each record appears only once.

###  3. Filling Missing Values

Use techniques like:

* Mean/median replacement
* Using default values
* Removing rows if necessary

###  4. Standardization

Same format across all data:

* Dates
* Names
* Phone numbers
* Currency

###  5. Automated checks

Rules that alert teams when data has problems.



##  Real Company Examples

### **Banks**

### **E-commerce**

Wrong inventory data = selling out-of-stock products.

Data quality = business quality.
